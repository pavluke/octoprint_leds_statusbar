# OctoPrint LEDs Statusbar

[EN](README.md) | RU

Индикатор статуса 3D-принтера на адресной LED-ленте (WS2812B), управляемый ESP8266. Опрашивает [OctoPrint](https://octoprint.org/) REST API и показывает текущее состояние печати цветом, длиной полосы прогресса и миганием.

## Как это выглядит

| Статус                               | Цвет    | Поведение                                            |
| ------------------------------------ | ------- | ---------------------------------------------------- |
| Printing                             | Зелёный | Растущая полоса прогресса, яркая, с подсветкой краёв |
| Paused                               | Зелёный | Полоса прогресса, мигает, с подсветкой краёв         |
| Completed                            | Зелёный | Полная лента, мигает                                 |
| Idle                                 | Зелёный | Полная лента, приглушённая, без мигания              |
| Offline after error                  | Красный | Полная лента, яркая, мигает                          |
| Offline                              | Красный | Полная лента, приглушённая, без мигания              |
| Not connected (OctoPrint недоступен) | Красный | Полная лента, приглушённая, мигает                   |

При старте устройства, во время подключения к Wi-Fi, лента мигает синим от центра к краям.

## Железо

- ESP8266 (NodeMCU/Wemos D1 mini и т.п.)
- Адресная LED-лента WS2812B, по умолчанию 23 диода (см. `NUM_LEDS` в `octoled.ino`)
- Данные ленты подключены к пину `D4`/`GPIO2` (см. `LED_PIN` в `octoled.ino`)

## Структура проекта

| Файл                | Назначение                                                                                      |
| ------------------- | ----------------------------------------------------------------------------------------------- |
| `octoled.ino`       | Основная логика: опрос OctoPrint, отрисовка статуса на ленте                                    |
| `defines.h`         | Enum статусов принтера и разбор ответа OctoPrint API                                            |
| `secrets.h`         | Твои личные данные подключения (Wi-Fi, OctoPrint) — **не коммитится**                           |
| `secrets.h.example` | Шаблон для `secrets.h` — пустые поля, безопасно коммитится                                      |
| `errors.h`          | Проверка, что все поля `secrets.h` заполнены; останавливает сборку с понятной ошибкой, если нет |
| `logger.h`          | Логирование по уровням (DEBUG/INFO/WARNING/ERROR)                                               |

## Настройка

1. Открой `secrets.h` и заполни все поля своими значениями:

   - `SECRET_WIFI_SSID`, `SECRET_WIFI_PASSWORD` — данные твоей Wi-Fi сети.
   - `SECRET_OCTOPRINT_IP` — IP-адрес сервера OctoPrint в локальной сети (4 числа через запятую).
   - `SECRET_OCTOPRINT_PORT` — порт OctoPrint (обычно `5000` или `80`).
   - `SECRET_OCTOPRINT_APIKEY` — Global API Key из OctoPrint: _Settings → API → Global API Key_.

2. Если что-то забудешь заполнить, сборка остановится с понятной ошибкой компиляции, указывающей конкретное поле — это делает `errors.h`.

3. Установи зависимости в Arduino IDE / PlatformIO:

   - [FastLED](https://github.com/FastLED/FastLED)
   - [OctoPrintAPI](https://github.com/stefanboehm/OctoPrintAPI)
   - [ESP8266 board support](https://github.com/esp8266/Arduino)

4. Залей `octoled.ino` на устройство.

## Логирование

По умолчанию в Serial-порт (115200 baud) пишутся сообщения уровня `INFO` и выше. Чтобы изменить уровень — например, для отладки — добавь перед `#include "logger.h"` в `octoled.ino`:

```cpp
#define LOG_LEVEL LOG_LEVEL_DEBUG  // максимальная детализация
```

Доступные уровни: `LOG_LEVEL_DEBUG`, `LOG_LEVEL_INFO`, `LOG_LEVEL_WARNING`, `LOG_LEVEL_ERROR`, `LOG_LEVEL_NONE` (полностью отключить). Сообщения ниже установленного уровня не попадают в скомпилированный бинарник — это не runtime-фильтр, а отбрасывание кода на этапе компиляции.

## Безопасность

`secrets.h` содержит реальный пароль Wi-Fi и API-ключ OctoPrint — не публикуй его и не коммить в git (он уже добавлен в `.gitignore`). Если ты делишься этим проектом, отправляй `secrets.h` отдельно от репозитория.
